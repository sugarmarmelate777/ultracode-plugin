# Contributing to Ultracode

> **Примечание:** Этот файл является переводом. Английская версия `CONTRIBUTING.md` является канонической при любых расхождениях.

Спасибо за интерес к проекту! Вот как вы можете помочь:

## 🐛 Нашли баг в навыке?

1. Откройте Issue на GitHub
2. Укажите, какой навык (`skills/<name>/SKILL.md`) содержит ошибку
3. Опишите конфликт или проблему (например, "навык X противоречит навыку Y")

## ✨ Хотите добавить новый навык?

1. Форкните репозиторий
2. Создайте папку `skills/<your-skill-name>/`
3. Внутри создайте `SKILL.md` по шаблону:

```markdown
---
name: Название навыка
description: Краткое описание в одну строку.
---

# Название навыка

## Directives

1. **Правило 1:**
   - Конкретная инструкция для ИИ.

2. **Правило 2:**
   - Конкретная инструкция для ИИ.
```

4. Обновите `LLMS.md` — добавьте навык в правильное место в workflow
5. Откройте Pull Request

## ⚠️ Правила для новых навыков

- Навык НЕ должен конфликтовать с существующими (проверьте `LLMS.md`)
- Навык НЕ должен ослаблять безопасность (Security Guard, Immutable Core)
- Навык должен быть **универсальным** — без привязки к конкретной ОС, IDE или API
- Пишите директивы как чёткие команды для ИИ, а не как рекомендации
- **НЕ** добавляйте секцию `**Context:**` — используйте только YAML frontmatter + `## Directives`
- Все проверки с участием пользователя (одобрения, вопросы) автоматически перехватываются глобальным CI/CD Gate. Не нужно добавлять ветки `if CI=true` внутрь навыков.
