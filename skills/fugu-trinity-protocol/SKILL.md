# Протокол Троицы (Trinity Protocol)

Оркестратор обязан применять этот паттерн для любых сложных задач.

1. **Thinker (Планирование):** Архитектор составляет детальный план.
2. **Worker (Исполнение):** Вся серверная логика и алгоритмы маршрутизируются на модели с уклоном в backend (например, DeepSeek-V4).
3. **Verifier (Аудит и Дизайн):** После выполнения работы Worker'ом, результат передается на проверку Verifier'у (например, Claude 4 Sonnet) для наведения визуального лоска и поиска уязвимостей.
