# Claude Design Sync (Two-Way UI Sync)

**Type**: `Frontend Development / UI Protocol`
**Trigger**: Когда пользователь просит "сверстать", "создать UI компоненты", "интегрировать дизайн" или работает над React-фронтендом.

## Protocol
Вместо того чтобы писать UI вслепую, агент использует возможности визуального редактора и дизайн-системы:

1. **Design System Init**: 
   Агент инициирует команду (или промпт) для парсинга локального кода и создания дизайн-системы в Cloud Design: `design-sync init` (экспорт текущих цветов, отступов и шрифтов).
2. **Pull UI Prototypes**:
   Когда пользователь генерирует UI в визуальном редакторе (Cloud Design) и предоставляет ID или код экспорта (HTML/ZIP), агент скачивает этот артефакт.
3. **Component Injection**:
   Агент переводит сгенерированный (или скачанный) HTML/React код в стандарты проекта:
   - Применяет локальные утилитарные классы (Tailwind).
   - Выносит переиспользуемые элементы в отдельные файлы (например, `src/components/ui/Button.tsx`).
   - Сохраняет логику "светлая/темная тема" (`dark mode`), протестированную в визуальном редакторе.
4. **Push UI Updates**:
   Любые изменения, внесенные агентом в локальный код, должны быть оформлены так, чтобы их можно было легко импортировать обратно в Cloud Design для визуальной проверки пользователем.
