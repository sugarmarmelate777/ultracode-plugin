# Side-by-Side Arena Generation

**Type**: `Generation Strategy / Swarm Tactic`
**Trigger**: Генерация креативов, лендингов, сложного UI, копирайтинг.

## Concept
Вместо того чтобы полагаться на один "нулевой" (zero-shot) ответ модели, агент оркеструет "Арену" (по аналогии с LMSYS Chatbot Arena).

## Execution Flow
1. **Parallel Generation:**
   Оркестратор запускает 2-3 параллельных субагента. Каждому дается одинаковый промпт, но:
   - Разная температура (например, `0.2`, `0.7`, `1.0`).
   - ИЛИ разные модели (например, Sonnet 3.5, GPT-4o, Llama 3 70B).
2. **Evaluation (The Judge):**
   После того как субагенты сгенерировали артефакты (например, 3 версии HTML/Tailwind лендинга), 4-й субагент (Judge) оценивает их по критериям:
   - Эстетика и "WOW-эффект" (см. Web Graphics).
   - Конверсионность (UI/UX).
   - Читабельность и адаптивность.
3. **Selection:**
   Judge выбирает лучший вариант, отбрасывает остальные, и Оркестратор внедряет именно его в кодовую базу.

Это критически улучшает качество выдачи, так как LLM отлично умеют сравнивать готовые результаты, даже если сами иногда ошибаются при первой генерации.
