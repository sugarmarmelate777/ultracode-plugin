# Архитектор GitHub Actions

## 1. Концепция Фабрики Экшенов
Вместо того чтобы держать все возможные `GitHub Actions` включенными в одном проекте, Ultracode использует **Библиотеку Шаблонов** (`actions-library`). Вы (ИИ-агент) выступаете в роли архитектора, который собирает нужные процессы под конкретную задачу.

## 2. Развертывание Экшенов
Когда пользователь просит создать новый проект, настроить CI/CD или обеспечить безопасность облачной инфраструктуры, вы должны скопировать нужные `.yml` шаблоны из ядра плагина в рабочую директорию проекта пользователя.

**Целевая директория в рабочем проекте:** `.github/workflows/`

**Доступные шаблоны в `ultracode-plugin/actions-library/`:**
1. `security.yml` — Для проектов, требующих высокой степени защиты (поиск утекших паролей, сканирование кода).
2. `quality-gate.yml` — Линтинг и проверка стандартов кода с помощью GitHub Super-Linter. Подходит для командной разработки.
3. `osint-cron.yml` — Шаблон ежедневного скрипта, запускающегося по крону для сбора данных, парсинга или аналитики.
4. `issue-ops.yml` — Управление процессами или агентами прямо из комментариев к задачам (Issue) на GitHub.

## 3. GitHub Apps (Инструкция для пользователя)
GitHub Apps нельзя установить программно (требуется клик владельца). Если проекту нужны мощные интеграции, вы должны выдать пользователю ссылки на маркетплейс и проинструктировать его нажать "Install":
- **Vercel / Netlify:** Для фронтенд-проектов (Next.js, React).
- **Dependabot:** Включить в настройках репозитория для авто-обновления библиотек.
- **Codecov:** Для визуализации покрытия тестами.
- **Sweep AI / Renovate:** Для автономного написания кода и закрытия мелких багов.
