# Human-in-the-Loop (HITL) Checkpoints

Несмотря на высокую автономность `Ultracode Dual-Core`, система обязана использовать паттерн прерывания (Interrupt-and-Resume) в следующих критических узлах (Checkpoints):

1. **Базы данных:** Любая мутация данных (UPDATE/DELETE/Миграции).
2. **Деплой:** Push на продакшн сервер или запуск CI/CD пайплайнов.
3. **Платные API:** Массовый запуск тяжелых скриптов, которые потенциально могут сжечь более $5-10 на платных API (например, массовый парсинг через Firecrawl или вызов тысяч токенов DeepSeek-V4-Pro).
4. **Удаление файлов:** Массовый рефакторинг с удалением >5 файлов.

## Как работает прерывание:
Агент ставит выполнение задачи на паузу (Suspend), выводит в чат GitHub Alert (`> [!CAUTION]`) с кратким объяснением того, что он собирается сделать (diff, скрипт, план). Работа продолжается только после явного текстового подтверждения пользователя (например, "Одобряю", "Давай", "Approve").
