# Claude Router Rule

**Type**: `Context Management / Memory`
**Trigger**: Обновление глобального стейт-файла (например, `.agent_state.md`, `CLAUDE.md`, `.workspace_state.md`).

## The Problem
Если агент постоянно дописывает логи и заметки в один глобальный файл (который читается при каждом старте), файл разрастается, забивает контекстное окно и сжигает токены.

## The Rule
1. **Max 200 Lines**: Главный файл состояния (`.agent_state.md` или `CLAUDE.md`) **НИКОГДА** не должен превышать 200 строк.
2. **Router Pattern**: Этот файл должен служить исключительно "Оглавлением" (Router). В нем запрещено хранить детали. Он должен содержать только ссылки на другие файлы памяти.
   *Пример:*
   - [x] Backend Arch: `docs/backend.md`
   - [x] API Contracts: `docs/api.md`
   - [ ] Current Tasks: `docs/tasks.md`

## Enforcement
Перед сохранением информации в глобальный стейт, агент должен:
1. Записать детали в специализированный лог-файл (например, `.ultracode/memory/task_42.md`).
2. Добавить или обновить ОДНУ строчку со ссылкой в `CLAUDE.md`.
3. Если `CLAUDE.md` превышает 200 строк, агент ОБЯЗАН провести рефакторинг: сгруппировать старые записи и вынести их в `docs/archive.md`.
