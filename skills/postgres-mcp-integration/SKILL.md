# PostgreSQL MCP Server Integration

Этот навык регламентирует работу подагентов с базами данных PostgreSQL посредством MCP.

## Правила работы с БД (Read-Only Default)
1. **Schema Discovery:** Прежде чем писать логику для backend, агент должен запросить схему БД через MCP (список таблиц, ключи, индексы), чтобы избежать галлюцинаций.
2. **Анализ данных:** Разрешено выполнять SELECT-запросы с LIMIT (не более 100 строк) для понимания структуры реальных данных.

## Деструктивные операции (Write/Drop)
Любые команды `INSERT`, `UPDATE`, `DELETE`, `DROP` или выполнение миграций **категорически заблокированы** для автоматического выполнения. Оркестратор обязан сначала сгенерировать SQL-скрипт, показать его пользователю и запросить одобрение через механизм `hitl-checkpoints`.
