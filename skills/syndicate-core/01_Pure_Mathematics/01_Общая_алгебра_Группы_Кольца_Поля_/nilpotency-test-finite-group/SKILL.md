# Nilpotency Test for Finite Groups

### Purpose

Determine if a finite group \(G\) is nilpotent and, if so, its nilpotency class.
Nilpotency means the lower central series \(\gamma_1 = G,\; \gamma_{i+1} = [\gamma_i, G]\) reaches the trivial subgroup.

### Algorithm

1. **Input:** A finite group \(G\), given either by a multiplication table, a list of permutations, or a finite presentation.
2. **Initialize:** Set \(H = G\), \(i = 1\).
3. **Iterate:**
   - Compute the commutator subgroup \(K = [H, G]\) as the subgroup generated by all commutators \([h,g] = h^{-1}g^{-1}hg\) with \(h \in H,\; g \in G\).
   - If \(|K| = |H|\), stop; the series has stabilised.
   - If \(|K| = 1\), output **nilpotent** with class \(i\).
   - Otherwise set \(H \leftarrow K\), \(i \leftarrow i+1\) and repeat.
4. **Decision threshold:** After stabilisation, if the final subgroup is not the identity, output **not nilpotent**.

### Computational Remarks

- For large groups, use a library that supports fast commutator subgroup calculation (e.g., GAP, SageMath).
- In practice, if the group is a \(p\)-group, nilpotency is guaranteed; the test still works but can be short-circuited.
- The algorithm always terminates because the orders strictly decrease until stabilisation or reaching 1.
- Example: \(D_8\) (dihedral of order 8) has lower central series: \(D_8 > C_2 > 1\), so nilpotent of class 2. \(S_3\) has series \(S_3 > A_3 > A_3\) -> not nilpotent.
